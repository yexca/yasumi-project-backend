# Yasumi Backend Documents

This directory holds backend-local documentation for two different jobs:

- `developer/`
  - the current and future backend development guide
- `original/`
  - historical development records kept for traceability

## Start Here

For current backend work, start with:

1. `developer/README.md`
2. the matching `developer/concepts/`, `developer/how-to/`, or `developer/reference/` document for your task

Use `original/` only when you need historical context, round plans, acceptance records, or archived release notes.

## Structure

- `developer/`
  - durable backend guidance organized by the reader's problem
- `original/`
  - archived round records and historical delivery notes

## Rules

- Do not store real secrets, tokens, personal data, or production connection details in documents.
- Historical archive documents should only receive path fixes, redaction, errata, or index adjustments.
- Current backend rules should be extracted into `developer/` instead of being left only in round documents.
