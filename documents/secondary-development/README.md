# Secondary Development Guide

This directory is for backend developers who extend the current service while keeping the codebase cohesive, loosely coupled, and lightweight.

## Entry Points

- `architecture-and-extension.md`
  - Current backend structure, dependency direction, and common extension paths.
- `contribution-guide.md`
  - Commit format, module naming, and minimum checks before submitting changes.

## Scope

- New authenticated endpoints.
- Sync upload capability changes.
- Authentication, configuration, migration, and telemetry adjustments.
- Documentation and maintainability improvements.

Round-specific implementation guidance lives in `../development-rounds/round-02-real-sync-and-structure/` while Round 02 is active.
